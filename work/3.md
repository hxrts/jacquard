# Lean Phase 3

This document captures the next formalization phase after the current destination-local field model, reduced private protocol, and reduced adequacy bridge.

The goal of this phase is to extend the proof story outward in the right order:

- from one local observer-controller state to a small network model
- from purely local public projection to router-facing publication semantics
- from observational local claims to reduced admission and installation semantics
- from local structural theorems to network-level safety theorems

The goal is not to jump immediately to full async transport or full-system optimality. The goal is to add the next semantic layers without collapsing the proof boundaries that are currently clean.

## Scope

In scope:

- a reduced finite network-of-nodes semantic layer
- node-local field state indexed by node and destination
- an explicit neighbor relation and round-based message movement
- router-facing publication semantics
- reduced route admission and installation semantics
- network-level safety theorems

Out of scope:

- full production async runtime semantics
- full transport verification
- full router implementation verification
- end-to-end convergence proofs for the production system
- global routing optimality proofs

## Success Criteria

- There is a finite network model above the current local field model.
- Router-facing publication is modeled explicitly and separately from local observational projection.
- Reduced route admission and installation semantics exist as a distinct proof layer.
- There is at least one nontrivial network-level safety theorem.
- The new network layer preserves the existing ownership boundary:
  - local field model owns local evidence processing
  - protocol layer owns cooperative private exchange
  - router-facing layer owns canonical publication and installation semantics

- [x] Define the global network state as a finite collection of node-local states plus the neighbor relation and any round-level message buffer needed for the reduced semantics.

  - local public corridor projection
  - sender

- [x] Add a short design note in the code explaining why the first network layer is intentionally round-based instead of async.

## Phase 2: Define Router-Facing Publication Semantics

- [x] Add a new router-facing boundary module, likely under `verification/Field/Router/`.
- [x] Define the publication boundary explicitly:
  - what a node may publish from its local `CorridorEnvelopeProjection`
  - what remains merely observational
  - what becomes router-facing candidate material
- [x] Keep publication semantics distinct from local projection semantics. Do not treat local projection as already canonical.
- [x] Define the smallest router-facing published object that is worth proving against.
- [x] State explicit publication honesty conditions:
  - explicit-path publication requires explicit-path-capable local state
  - publication support cannot exceed local support
  - publication hop bounds must remain within the local projection envelope
- [x] Prove at least one theorem showing publication is conservative with respect to local state.
- [x] Update the documentation note in `verification/Field/Docs/Model.md` or add a new doc if the publication boundary deserves its own note.

## Phase 3: Add Reduced Admission Semantics

- [x] Define a reduced admission layer for router-facing field candidates.
- [x] Keep the first admission semantics narrow and declarative:
  - candidate observed
  - candidate admitted
  - candidate rejected
- [x] Do not model the whole production router. Model only the proof-relevant admission boundary.
- [x] State the minimum admission predicates needed to support safety theorems:
  - candidate shape well-formedness
  - publication honesty preconditions
  - local-to-admission compatibility
- [x] Prove at least one theorem of the form:
  - admitted candidates satisfy publication honesty
  - or admission cannot strengthen a claim beyond the candidate that was published
- [x] Keep the boundary explicit that admission is the first place observational field output can become eligible for canonical handling.

## Phase 4: Add Reduced Installation Semantics

- [x] Define a reduced installation layer above admission.
- [x] Keep the installed route object very small and canonical:
  - destination
  - supporting node or path class
  - reduced route shape
  - any proof-relevant installed confidence or support marker
- [x] State explicitly that installation semantics are canonical-control-plane semantics, not local observational semantics.
- [x] Prove at least one theorem that installation preserves admitted-route honesty.
- [x] Prove at least one theorem that installation cannot occur without an admitted candidate.
- [x] Keep this layer small. Do not import full router lifecycle machinery yet.

## Phase 5: Prove Network-Level Safety Theorems

- [x] Identify the strongest network-level safety statements that are actually true in the reduced synchronous network model.
- [x] Prove at least one theorem of the form:
  - no spontaneous explicit-path installation
  - or no canonical route installation without an honest published candidate
  - or network-wide public claims remain conservative with respect to node-local evidence
- [x] Prove at least one theorem relating the network layer to the existing local model:
  - local honesty lifts to network publication honesty
  - or local support bounds lift to network publication support bounds
- [x] Prefer safety theorems over liveness or optimality theorems at this stage.
- [x] Keep theorem statements explicit about which layer owns which fact:
  - local evidence
  - publication
  - admission
  - installation

## Phase 6: Prepare For Async Later Without Encoding It Now

- [x] Add a short design note or API comment stating what parts of the new network model are deliberately chosen so that async transport can be added later.
- [x] Make sure message delivery is factored cleanly enough that a later async semantics could replace the round buffer without rewriting the local field model.
- [x] Avoid baking fairness, timing, or transport-loss assumptions into the synchronous network core unless they are explicitly named as assumptions.
- [x] Add one short note about which future async concerns will belong in:
  - protocol layer

## Roadmap

This is the longer-range roadmap for building out the full proof stack from the current local model to a much richer field system.

### Stage 1: Local And Reduced Protocol Foundations

- [x] Maintain and refine the local observer-controller model.
- [x] Strengthen the information layer with richer probabilistic objects and sharper blindness theorems.
- [x] Keep the private protocol Telltale-shaped and continue replacing field-local glue with direct family instantiations where possible.
- [x] Strengthen the reduced runtime adequacy bridge.

### Stage 2: Network And Control-Plane Boundaries

- [x] Add the reduced finite network semantic layer.
- [x] Add router-facing publication semantics.
- [x] Add reduced admission semantics.
- [x] Add reduced installation semantics.
- [x] Prove network-level safety theorems before any liveness or optimality claims.

### Stage 3: Stronger Runtime And Telltale Reuse

- [x] Strengthen receive refinement into deeper subtype-replacement reuse.
- [x] Connect reduced protocol traces more directly to Telltale protocol-machine adequacy families.
- [x] Improve replay-visible semantic object and observer-projection proofs.
- [x] Move from reduced simulation witnesses toward stronger runtime refinement theorems.

### Stage 4: Richer Information And Quantitative Results

- [x] Strengthen the belief object and uncertainty functionals.
- [x] Add stronger entropy, divergence, and information-loss theorems.
- [x] Add stronger temporal stabilization theorems over repeated rounds.
- [x] Connect the local field model more directly to the classical-analysis style API/instance split used in Telltale.

### Stage 5: Async And End-To-End System Semantics

- [x] Add a first reduced asynchronous network/transport semantics.
- [x] Make message delivery, loss, delay, and retry assumptions explicit.
- [x] Relate the async network model to the reduced synchronous network model where possible.
- [x] Extend adequacy toward a stronger Rust-runtime correspondence theorem.

### Stage 6: Larger-System Claims

- [x] Add first async publication-safety and observer-surface theorems over the reduced async model.
- [x] Add first system-level statistical summaries and proof-boundary markers above the async model.

Later research horizon:

- Convergence or stabilization theorems belong after the async semantics are richer than the current reduced queue model.
- Mean-field or fluid-limit results belong only after the finite network and async layers stabilize enough to support them honestly.
- Global routing quality or optimality claims belong only if a later semantic model is genuinely strong enough to justify them.

The intended build order is:

1. local semantics
2. information layer
3. private protocol
4. reduced adequacy
5. reduced network
6. publication/admission/installation
7. network safety
8. stronger Telltale/runtime reuse
9. async semantics
10. larger-system convergence and quantitative results
